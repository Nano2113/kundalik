# Электронный журнал — FULLSTACK (frontend + backend)

Это готовый полный проект:
- **Frontend**: `public/` (HTML/CSS/JS) — удобный на телефоне (responsive)
- **Backend**: Node.js + Express (`server.js`)
- **DB**: SQLite (`database.sqlite` создаётся автоматически)

## Входы (по умолчанию)
**Админ**: `Nodir` / `Nodir228`  
**Учителя**: `admin1..admin10` / `admin1pass..admin10pass`  
(у каждого учителя есть **один предмет**, админ видит всё)

## Возможности
- Учитель видит **только свой предмет**
- Админ может: добавлять классы, добавлять/удалять учеников, снимать блокировку
- Оценки: **1–10** или **П/О/Н**
- Если у ученика **П/Н > 2** (по всем предметам) → ученик **блокируется** (blacklist)
- Учителя не могут ставить/менять оценки заблокированным
- Админ может снять блокировку в админ-панели
- При изменении уже существующей оценки нужен комментарий

## Запуск локально
1) Установите Node.js (LTS)
2) В папке проекта:
```bash
npm install
npm start
```
Откройте:
`http://localhost:3000`

## Как залить на хостинг
GitHub сам по себе **не запускает Node.js**.
Вы делаете так:
- Код кладёте в GitHub
- Потом деплоите на **Render / Railway / Fly.io / VPS** (backend)
- Домен/ссылка будет оттуда

### Render (самый простой вариант)
- Create New → Web Service
- Connect GitHub repo
- Build Command: `npm install`
- Start Command: `npm start`
- Добавьте переменную окружения `JWT_SECRET` (любая длинная строка)

## Где менять логины/пароли/предметы
- В базе `database.sqlite`, таблица `users`
- Или меняйте в `server.js` в функции `seed()` и удалите `database.sqlite`, чтобы сидер создал всё заново.
